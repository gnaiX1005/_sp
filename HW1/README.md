## 第一階段：編譯（Translation / Parsing）

當你執行 `./my_compiler test.c` 時，程式會先進行翻譯工作，將高階語言轉化為**四元組 (Quadruples)**。

### 1. 詞法分析 (Lexer)

* 程式會讀取 `test.c` 的字串。
* `next_token()` 函式會像切割機一樣，把字串切成一個個有意義的標記（Tokens）。例如：把 `while` 辨識為關鍵字，把 `sum` 辨識為 ID。
* **關鍵修正**：它現在學會了跳過 `#` 開頭的行，避免被 `#include` 混淆。

### 2. 語法解析 (Parser) - 建立步驟清單

* **賦值語句**：看到 `i = 0`，會生成 `IMM 0 - t1` (把 0 放到暫存器) 和 `STORE t1 - i` (把暫存器存入變數 `i`)。
* **While 迴圈（核心邏輯）**：
1. 先標記目前的地址（`start_pc`），這是等一下要跳回來重新檢查條件的地方。
2. 解析括號內的條件（如 `i < 10`），生成 `CMP_LT`。
3. 生成 `JMP_F`（跳轉如果為假）：這是一個**「伏筆」**，如果條件不成立，就跳到還不知道在哪裡的「結束位置」。
4. 解析大括號內的內容。
5. 生成 `JMP`：解析完畢後，強制跳回第一步的 `start_pc` 重新檢查。
6. **回填 (Backpatching)**：把剛才 `JMP_F` 的「問號」填上現在的地址，這樣條件失敗時才知道要跳去哪。

## 第二階段：虛擬機執行 (Virtual Machine)

編譯完成後，會得到一份 `quads` 陣列（指令清單），接著交給 `vm()` 函式。

### 1. 記憶體管理 (Stack & Frame)

* **全域空間**：`stack[0]` 是你的全域變數區。
* **變數存取**：`set_var` 會在目前的空間尋找變數，找不到就新建；`get_var` 則會讀取數值。

### 2. 指令迴圈 (The Fetch-Execute Cycle)

虛擬機使用一個 `pc` (Program Counter) 指標，從 `000` 號指令開始抓取並執行：

* **算術指令** (`ADD`, `SUB`, `MUL`, `DIV`)：從當前 Frame 抓出數值，計算後存回暫存變數。
* **條件判斷** (`CMP_LT` 等)：比較兩個數值，結果存入暫存變數（`1` 為真，`0` 為假）。
* **跳轉指令**：
* `JMP_F`：檢查條件變數。如果是 `0`，就把 `pc` 直接修改成目標地址。
* `JMP`：無條件將 `pc` 修改成目標地址。


* **函數呼叫**：如前所述，會 `sp++` 開闢新的空間，執行完後再 `sp--` 回來。

---

## 3. 執行結束

當 `pc` 超過最後一條指令時，迴圈停止。
最後程式會遍歷 `stack[0]`（全域變數空間），將所有不是 `t` 開頭的變數名稱與數值印出來。這就是為什麼你能看到 `>> sum = 45` 的原因。

---

### 舉例說明 `while` 的執行動線

假設指令如下：

* `002: CMP_LT i 10 t2` (檢查 i < 10)
* `003: JMP_F t2 - 007` (若為假，跳到 007)
* `004: ... (迴圈主體)`
* `006: JMP - - 002` (跳回 002)
* `007: ... (結束後的第一條指令)`

**執行路徑：**

1. 執行 `002`，發現 `i` 是 0，`t2` 變成 1。
2. 執行 `003`，因為 `t2` 不是 0，所以不跳轉，繼續執行 `004`。
3. 執行完 `005` 來到 `006`，`pc` 被強制改回 `002`。
4. 重複直到 `i` 變成 10。
5. 再次回到 `002`，`t2` 變成 0。
6. 執行 `003`，發現 `t2` 是 0，`pc` 被改為 `007`，成功跳出迴圈！
